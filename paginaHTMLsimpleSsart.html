ARQUITECTURA WEB Y FRONTEND

Esta pagina presenta una vision clara de arquitectura web y desarrollo frontend para comprender como se construye una aplicacion moderna desde la base.

Que es la arquitectura web

La arquitectura web define como se coordinan clientes, servicios, servidor y base de datos para responder de forma eficiente y segura a cada solicitud. Permite que el navegador del usuario se comunique con servidores y bases de datos.

Componentes de la arquitectura web

  • Cliente: navegador del usuario que solicita informacion al servidor.
  • Servidor: maquina que procesa solicitudes y envia respuestas.
  • Base de datos: lugar donde se almacenan los datos de la aplicacion.

Tabla de ventajas y desventajas

Caracteristicas Ventajas Desventajas
Acceso y usabilidad Acceso desde navegador y dispositivo sin instalacion. Dependencia de conexion a internet.
Escalabilidad y rendimiento Posibilidad de escalar para manejar mayor trafico. Puede degradarse si hay sobrecarga de servidor.
Seguridad Actualizaciones centralizadas. Mayor exposicion a ataques si no se protege bien.

Que es el desarrollo frontend

El frontend se enfoca en disenar e implementar la interfaz con la que interactua el usuario, priorizando usabilidad, claridad y adaptacion a distintos dispositivos. Es lo que el usuario ve e interactua al navegar por internet.

Tecnologias del desarrollo frontend

  1. HTML: estructura del contenido.
  2. CSS: estilos y diseno visual.
  3. JavaScript: interactividad y comportamiento dinamico.

Tabla de ventajas y desventajas frontend

Caracteristicas Ventajas Desventajas
Desarrollo y UX Resultados visuales inmediatos y enfocados en experiencia. Mantenimiento continuo y actualizaciones frecuentes.
Interaccion Curva de aprendizaje inicial amigable. Cambios tecnologicos muy rapidos.
Diseno y plataforma Facil crear interfaces responsivas para movil y escritorio. Retos de compatibilidad entre navegadores.

Ampliacion Ssart: flujo de una peticion web

  1. Usuario abre una URL en el navegador.
  2. Navegador envia solicitud HTTP al servidor.
  3. Backend procesa reglas y consulta base de datos.
  4. Servidor responde con HTML, CSS, JS y datos.
  5. Frontend renderiza la interfaz para el usuario final.

Buenas practicas extra

  • Separar estructura, estilos y logica.
  • Usar etiquetas semanticas para accesibilidad.
  • Aplicar diseno responsive desde etapas iniciales.
